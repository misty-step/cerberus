Backlog 016 | Peer Harness Boundary

Run fixture-backed peer commands before spending provider budget.

The execution plan now makes the command inspectable. The next step is a bounded live runner: render the prompt, invoke the configured command, capture the transcript, parse the marked artifact, and keep paid provider profiles behind explicit budget acknowledgement plus non-argv prompt transport.

Target Outcome

`CERBERUS_PEER_HARNESS_LIVE=1 cerberus-peer-harness` can execute a fixture peer command and emit a completed `ReviewerArtifact.v1` from the captured transcript.

Chosen Design

Extract one bounded subprocess primitive from the adapter crate, then let the peer runner own only prompt rendering, budget gating, transcript capture, and artifact parsing.

Delete-First

No new provider client, retry engine, scheduler, queue, or harness wrapper. The existing command profile contract is enough for this slice.

Stop Rule

Stop when fixture live execution, timeout containment, provider-budget and provider-argv refusal, docs, full gates, and fresh critic review are green.

Quality System

Part Decision
Claim Peer commands can run through the configured protocol without provider spend or unbounded child processes.
Falsifier A paid profile executes without budget acknowledgement, a provider prompt travels through argv, timeout leaves a child alive, transcript is not captured, or malformed output becomes an artifact.
Driver Fixture live profile plus `CERBERUS_PEER_HARNESS_LIVE=1` CLI invocation.
Grader Artifact schema validation, transcript marker inspection, timeout tests, and full Rust/legacy/static gates.
Critic Fresh-context review gets only the diff, this oracle, and the works-critique card.

Commands

cargo test -p cerberus-adapter command_harness
cargo test --workspace peer_harness_live

mkdir -p tmp
CERBERUS_PEER_HARNESS_LIVE=1 cargo run --locked -p cerberus-cli --bin cerberus-peer-harness -- \
  --harness fixture-live \
  --profiles fixtures/harnesses/live-peer-command-profiles.json \
  --input fixtures/harnesses/peer-runner-input.json \
  --output tmp/peer-runner-live-artifact.json \
  --transcript-output tmp/peer-runner-live-transcript.txt
cargo run --locked -p cerberus-cli -- validate tmp/peer-runner-live-artifact.json
