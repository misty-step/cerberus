Backlog 006 / No-Spend Evidence Hygiene

Make eval output directories self-contained after every rerun.

Selected reruns are now a normal part of hardening Goose, OpenCode, Pi, OMP, GLM, Kimi, and DeepSeek cells. A reused output directory must not keep stale generated transcripts, inputs, artifacts, or plans that can mislead the next operator.

Chosen Design

Delete generated eval paths before each run. `eval-harness` clears only its known generated outputs: `report.json`, `transcripts/`, `inputs/`, `artifacts/`, and `plans/`.

No manifest, planner, or semantic heuristic is needed. The current report remains the authority for the current cell set.

Stop rule: do not run provider-backed live eval without explicit budget acknowledgement.

Outcome

A selected no-spend eval rerun into a previously used output directory leaves no stale Cerberus-generated eval files from a broader or older run.

Standards

Keep the CLI behavior narrow and mechanical. Do not add a second evidence index. Do not spend providers. Do not change promotion semantics.

Proof

Focused regression test, reused-directory CLI QA, report validation, full repo gates, and fresh artifact-only criticism.

Verification System

Claim: an eval evidence packet represents the latest `eval-harness` run, not a mixture of current and stale generated files.

Falsifier: after a selected rerun, stale files still exist under `transcripts/`, `inputs/`, `artifacts/`, or `plans/`, or the current report is not schema-valid.

Driver: seed stale generated paths, run a selected offline eval, validate `report.json`, and inspect the file list.

Evidence: `tmp/evals/eval-output-hygiene-2026-06-19/` report, transcript, file list, and checksums.

Critic Focus

Attack destructive scope, symlink handling, live-peer artifact creation, report/transcript consistency, and whether this overclaims provider quality.

1. Red

Add a regression test that seeds stale generated eval files, reruns a selected offline eval into the same directory, and expects the stale paths gone.

2. Green

Clear only the known `eval-harness` generated top-level paths after creating the output directory and before producing the current report.

3. Docs

Update backlog 006 and the shaping note to make reused output directory behavior explicit for future staged harness/model reruns.

4. Gates

Run focused Rust tests, reused-directory CLI QA, validation commands, full workspace tests, legacy Elixir gates, shellcheck, and fresh critic review before commit.
