BACKLOG 005 / LEGACY SURFACE RETIREMENT

Move consumer workflow scaffolding behind a Rust CLI fixture without retiring the Node secret path yet.

The durable outcome is narrower than a full rewrite: Cerberus gets a Rust-owned, testable workflow scaffolder path for generated workflow parity, while the existing Node command remains the compatibility wrapper for interactive API-key capture and live GitHub secret configuration.

Chosen Design

Add cerberus-cli init-workflow for deterministic workflow creation and preservation decisions, with --report-out JSON evidence for QA.

Do not call gh secret set, read secrets, or remove bin/cerberus.js in this slice.

Quality System

Risk tier: substantive CLI/public workflow change. Proof: live CLI fixture run, focused Rust tests, aggregate repo gates, and fresh-context critic.

Standards

At this earlier slice, Rust started taking durable repo behavior while Node still covered unmatched live-secret compatibility. Later slices moved secret setup and retired the unpublished npm wrapper. The public workflow template must not drift.

Acceptance

Rust CLI creates the consumer workflow from templates/consumer-workflow-reusable.yml, reports up-to-date when equal, and preserves differing existing files.

Verify

cargo test -p cerberus-cli init_workflow, a live scratch repo CLI run, validate-retirement, node --check, and final gates.

Stop If

Secret setup must be implemented to make the command honest, generated workflow differs from the template, or the inventory would imply Node is retired.

Verification System

Claim: Rust can own the deterministic workflow-file half of cerberus init.

Falsifier: a blank repo does not get the template, equal files are rewritten, or differing files are overwritten.

Driver: fixture tests plus a scratch CLI run with --repo-root, --template, and --report-out.

Grader: file content equals the checked template and report status is created, up_to_date, or preserved_existing.

Evidence packet: tmp/init-workflow-2026-06-19/ report and generated workflow.

Delete-First

Requirement questioned: a full Rust interactive scaffolder is not needed to advance retirement evidence today.

Deleted or simplified: no hidden prompt, no gh wrapper, no new schema, no dependency.

Only then automated: workflow file planning is repeated, deterministic, and currently trapped inside the Node wrapper.

Execution

1 frame

Read the Node scaffolder, template, CLI command shape, retirement schema, and active 005 inventory.

Evidence: files read in this session; plan opened before edits.

2 build

Add the minimal Rust workflow planner and CLI command; test create, equal, and differing-file cases through public-ish functions.

Evidence: focused Rust tests fail if workflow parity or overwrite protection breaks.

3 document

Update README and retirement inventory to describe the Rust workflow path and keep Node compatibility pending for live secret configuration.

Evidence: checked inventory validation and docs no longer imply only Node can scaffold workflows.

4 review

Run live CLI QA, aggregate gates, and a fresh-context critic against only the diff plus this oracle.

Evidence: critic reports no blocking public-surface, compatibility, or verification gap.
