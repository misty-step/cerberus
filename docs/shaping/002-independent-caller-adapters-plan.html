BACKLOG 002

independent caller adapter contract

CERBERUS / CALLER BOUNDARY

Let Bitterblossom and Olympus call the same review contract without knowing each other.

Add a Rust adapter crate with fixture-backed request builders, caller-owned receipt/projection examples, and a static guard that rejects cross-caller references.

CHOSEN DESIGN

Build `cerberus-adapter` as a small SDK over `ReviewRequest.v1` and `ReviewRunArtifact.v1`; it owns caller contract examples, not event queues or GitHub posting policy.

PROOF

cargo test --workspace caller_contracts runs both caller fixtures through the fake core and the no-cross-caller guard.

STOP IF

A fixture needs Bitterblossom runtime code, Olympus Argus types, or caller-specific policy inside `cerberus-core`.

BITTERBLOSSOM

Event plane: task, trigger, run ledger, budgets, retries, receipts. Cerberus returns a review artifact it can store under its ledger.

OLYMPUS ARGUS

Posting policy owner: activation gate, stale-head suppression, marker dedupe, severity caps, and GitHub review creation stay outside Cerberus.

CERBERUS

Request/artifact core plus adapter examples. No queue, no caller ledger, no posting authority, no sibling caller references.

IMPLEMENTATION FLOW

01

Adapter crate

Add builder helpers and caller receipt/projection structs that validate public schemas.

Keeps caller ergonomics outside `cerberus-core`.

02

Fixtures

Add Bitterblossom task and Olympus Argus request fixtures that invoke the fake core without external repos.

CI can prove the contract locally.

03

Boundary guard

Scan caller fixture text for forbidden sibling names, repo paths, and type names.

Prevents accidental caller coupling from entering the contract examples.
