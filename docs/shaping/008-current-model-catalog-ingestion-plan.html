BACKLOG 008

current model catalog ingestion

CERBERUS / EVAL EVIDENCE

Make model catalog facts executable evidence, not copied research prose.

Add a narrow CLI step that reads the checked eval matrix, ingests a raw OpenRouter-compatible catalog from file or URL, writes the raw JSON evidence, and emits a refreshed schema-valid matrix for the existing eval runner.

CHOSEN DESIGN

Keep catalog parsing in the CLI/operator layer. `cerberus-core` keeps grading eval cells; the CLI refreshes matrix model facts before execution.

QUALITY SYSTEM

Standards: source evidence is cached verbatim, derived candidates validate, missing catalog fields fail loudly, and production defaults remain unchanged.

STOP IF

The implementation needs paid model calls, provider credentials, a generic provider abstraction, or changes to legacy defaults.

INPUT

Existing `HarnessModelMatrix.v1` plus a raw OpenRouter-compatible catalog from a local fixture, cached download, or HTTPS source.

OUTPUT

Raw catalog JSON copied to an evidence path and a refreshed matrix containing updated `ModelCandidate.v1` rows with previous snapshots.

PROOF

The refreshed matrix validates and can drive `cerberus-cli eval-harness` without hand-editing model prices, context windows, or supported parameters.

IMPLEMENTATION FLOW

01

Fixture and parser

Add a minimal OpenRouter catalog fixture and parser tests for the models already in the matrix.

No network required for unit tests.

02

CLI command

Add `refresh-model-catalog` with `--matrix`, `--catalog-source`, `--out`, `--raw-out`, and optional `--observed-at`.

URL mode is an operator convenience; file mode is the deterministic gate.

03

Eval integration proof

Run the existing eval harness against the refreshed matrix and validate the report.

This proves catalog ingestion feeds the existing eval loop.
