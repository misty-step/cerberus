BACKLOG 001

Rust review engine contract implementation plan

CERBERUS / CONTRACT SPINE

Make `ReviewRequest + ReviewConfig -> ReviewRunArtifact` executable offline.

The first Rust milestone is a deep artifact core, not a service rewrite. It validates source-agnostic request fixtures, runs a deterministic fake reviewer harness, persists a review-run artifact, and renders markdown without GitHub credentials or network access.

CHOSEN DESIGN

Three crates: `cerberus-schema` owns versioned data contracts, `cerberus-core` owns review execution and aggregation, and `cerberus-cli` is the narrow operator surface.

QUALITY SYSTEM

Standards: source-agnostic inputs, artifact-first outputs, no GitHub/service coupling inside core crates, behavior-tested CLI surface.

Proof: backlog 001 CLI oracle, `cargo test --workspace`, legacy gates, fresh architecture/TDD critics.

Premise source: sha256:375022e89a124830363d7e6e4361b7d98d3f1ba4e3880a035df58285efc420ba backlog.d/001-rust-review-engine-contract.md

STANDARDS

Deep Rust modules, simple public interface, fixtures before behavior, deterministic fake reviewer first.

ACCEPTANCE

The four backlog 001 commands run locally and produce a schema-valid artifact plus markdown render.

VERIFY

`cargo test --workspace`; validate fixtures; review local diff fixture; render artifact; legacy gates.

STOP IF

Core needs GitHub, network, HTTP, queueing, live LLM credentials, or caller-specific runtime types.

CURRENT STATE

The repo has a legacy Elixir donor engine and no Rust workspace. Backlog 001 is `ready` and defines the offline CLI oracle.

CHANGE SHAPE

Add workspace, schemas, fixture set, fake runner, artifact persistence, markdown renderer, and inline-comment candidate projection. Leave adapters and live LLMs out.

PROOF SURFACE

The CLI itself is the live surface: validate, review, render. Unit tests support it but do not replace it.

IMPLEMENTATION PHASES

01

Contracts and fixtures

Define versioned Rust types and checked-in request fixtures for local diff, GitHub-shaped PR, clean change, and degraded reviewer run.

Tests assert fixture parsing through public schema APIs.

02

Fake review execution

Core consumes request/config and returns reviewer artifacts plus aggregated verdict. The fake harness is deterministic and fixture-driven.

No live model, no network, no adapter concerns.

03

CLI proof

`validate`, `review`, and `render` commands exercise the core path and write/read artifacts from disk.

This is the first operator-facing Rust surface.

04

Review and harden

Run fresh critics, legacy gates, CLI QA, and fix blockers before committing.

Reviewers get diff + oracle, not author reasoning.

ORACLE

cargo test --workspace
cargo run --locked -p cerberus-cli -- validate fixtures/review-request/*.json
cargo run --locked -p cerberus-cli -- validate fixtures/review-config/*.json
cargo run --locked -p cerberus-cli -- review --fixture fixtures/review-request/local-diff.json --out tmp/review-run
cargo run --locked -p cerberus-cli -- render tmp/review-run/review-run-artifact.json
cargo run --locked -p cerberus-cli -- render-comments tmp/review-run/review-run-artifact.json

RISKS

Spec-shaped code that validates JSON but cannot execute, shallow crates that only pass structs through, GitHub leakage into the core, tests that skip the CLI path, and fake reviewer behavior that hides degraded-run semantics.

REVIEW FOCUS

Attack the module boundary, fixture coverage, artifact stability, CLI workflow, and whether the first interface is simpler than the implementation.
