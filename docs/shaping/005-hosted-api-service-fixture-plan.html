Backlog 005 / hosted API service fixture / 2026-06-19

Freeze the hosted API response contract before replacing the server

POST ingress and request acquisition are covered. This slice proves the observable service responses for auth, health, queued creation, status reads, and store failures without starting a Rust server.

Target Outcome

cerberus-cli hosted-api-service-fixture renders Elixir-compatible HTTP status/body reports from explicit route, auth, request body, and store fixture inputs.

Design Choice

Keep compatibility rules in cerberus-adapter. The CLI only reads files and writes a report. No web framework, queue, SQLite clone, or deployment surface.

Proof Surface

Adapter tests, CLI fixture tests, live fixture outputs, token-leak grep, inventory validation, repo gates, and fresh public API critic.

Stop condition: done when health bypasses auth, non-health routes require the bearer token, POST store outcomes map to legacy bodies, GET status returns stored run JSON or not_found, and no token material appears in reports. HTTP server wiring and deployment remain pending.

Verification System

ClaimRust can render the hosted API's observable service contract from fixture inputs.
FalsifierAuth is bypassed for non-health routes, status bodies differ from legacy store shape, store failures map incorrectly, or fixture output leaks bearer/API token material.
Drivercerberus-cli hosted-api-service-fixture --method ... --path ... --store ... --out ...
GraderExact HTTP status/body assertions, token absence checks, and retirement inventory validation.
Evidence Packettmp/hosted-api-service-2026-06-19/
CadenceAfter implementation, after blocker fixes, and before commit.

Delete-First Check

  • Requirement questioned: replacing the web server now would add runtime surface before the response contract is fully frozen.
  • Deleted or simplified: no HTTP listener, no async queue, no SQLite compatibility layer, no deploy script.
  • Only then automated: one CLI fixture because the same response contract feeds dispatch polling, server replacement, and retirement evidence.
