Backlog 005 / hosted API HTTP service smoke / 2026-06-19

Put real HTTP around the hosted API fixture contract

Rust already renders hosted API service responses offline. This slice proves those responses can be served through a bounded local HTTP listener, creating the target that later deployment smoke can exercise.

Target Outcome

cerberus-cli hosted-api-serve-fixture binds a loopback address, writes a ready file, handles a bounded number of requests, and returns the existing fixture HTTP bodies.

Design Choice

Keep compatibility semantics in cerberus-adapter. Add a tiny CLI-owned listener with no new dependency, no async runtime, no queue worker, and no database layer.

Proof Surface

CLI integration tests spawn the server and make real TCP requests. A shell QA packet captures curl responses, token-leak checks, inventory validation, repo gates, and a fresh critic.

Stop condition: done when health, missing auth, queued GET, queued POST, and store-error POST are proven over HTTP. Do not claim Rust hosted API parity: real queue/store lifecycle, live GitHub acquisition, provider review execution, deployment, and cutover remain pending.

Verification System

ClaimRust can serve the fixture-backed hosted API compatibility contract over a real local HTTP socket.
FalsifierThe command cannot bind and publish its address, hangs after the bounded request count, returns report wrappers instead of HTTP bodies, leaks token material, or diverges from the existing fixture status/body contract.
Drivercerberus-cli hosted-api-serve-fixture --api-key ... --store ... --ready-file ... --max-requests ...
GraderExact HTTP status/body assertions, child process exit assertion, token absence grep, retirement inventory validation, and fresh public API critic.
Evidence Packettmp/hosted-api-http-service-2026-06-19/
CadenceAfter test red, after implementation, after any blocker fix, and before commit.

Delete-First Check

  • Rejected scope: production server framework, queue lifecycle, SQLite compatibility, deployment scripts, and live GitHub acquisition.
  • Kept surface: one bounded CLI fixture server because it creates an executable HTTP proof path for the already frozen contract.
  • Exit strategy: later replace this fixture server with the real Rust hosted API service once queue, store, and acquisition contracts are implemented.
