BACKLOG 007

rust harness runtime boundary

EXECUTION BOUNDARY

Let core aggregate reviewer artifacts from a harness interface, not hardcoded fake review calls.

Preserve deterministic local fixtures while creating the narrow Rust seam that future Pi, Goose, OpenCode, OMP, Sprites, and provider adapters can implement.

CHOSEN DESIGN

Add a `ReviewHarness` trait and default deterministic harness. `review` delegates through it; `review_with_harness` accepts custom runners.

PROOF

Focused tests for custom artifacts, degraded runner errors, existing fake fixture behavior, full workspace tests, and legacy gates.

STOP IF

The interface leaks GitHub, shell command execution, provider-specific prompts, or caller-owned runtime policy into `cerberus-core`.

CORE OWNS

Validation, artifact aggregation, dedupe, verdicts, coverage, reserves, and rendering.

HARNESS OWNS

How a reviewer artifact is produced: fake fixture, local CLI, Sprites lane, hosted model, or another adapter.

THIS SLICE

No live external command execution. The deliverable is the Rust boundary plus deterministic proof.
