Backlog 018

Run Cerberus eval cells through the live peer harness without spending provider budget by default.

The peer runner can now use private prompt files. The next step is to make the evaluation runner drive that live protocol and keep the proof packet: input, execution plan, transcript, artifact, and report cell.

Work Contract

Outcome: `eval-harness --execution-mode live-peer` emits a schema-valid report with `live_harness` cells from `cerberus-peer-harness`.

Design: core grades externally produced artifacts; CLI owns subprocess orchestration and evidence files.

Stop: no provider launch without budget acknowledgement and required env; no reviewer default change.

Verification System

  • Claim: live peer eval creates a valid report cell and durable evidence.
  • Falsifier: missing transcript/plan, offline pass, provider spend, or duplicated grading semantics.
  • Driver: `cerberus-cli eval-harness --execution-mode live-peer`.
  • Grader: report validation, transcript markers, focused Rust tests, provider gated QA.

Quality System

  • Substantive CLI/operator workflow change.
  • Proof: live fixture QA, structural gates, full repo gates, fresh critic.
  • Review lens: public CLI workflow, compatibility, operations, module depth.
  • Residual risk must name skipped paid provider bakeoff.
