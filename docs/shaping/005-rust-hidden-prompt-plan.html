BACKLOG 005 / NODE PROMPT RETIREMENT

Move source-checkout API-key prompting into Rust while npm setup remains Node-owned.

The previous slice moved noninteractive source-checkout setup into cerberus-cli init. This slice closes the Rust prompt parity gap: on an interactive Unix TTY, Rust prompts without echo, then sends the key to gh secret set through stdin. The npx @misty-step/cerberus init package path remains Node-owned until a later npm parity decision.

Chosen Design

Keep the public command unchanged. If CERBERUS_API_KEY is missing and --api-key-stdin is absent, init prompts only when stdin is a TTY.

Use Unix termios through the existing workspace libc dependency. Non-Unix or non-TTY environments fail closed with the existing stdin/env guidance.

Quality System

Risk tier: high-stakes CLI secret handling. Proof: focused Rust tests, PTY live QA, fake-gh capture, retirement inventory validation, aggregate gates, and fresh critic.

Standards

Rust owns setup logic. Secrets do not appear in argv, reports, committed fixtures, or PTY transcript output.

Acceptance

cerberus-cli init prompts on a local TTY, hides typed input, writes or preserves the workflow, and configures the repository secret through fake gh.

Verify

PTY driver sends a test key, transcript omits the key, fake gh stdin contains it, fake gh env is unset, and report stays redacted.

Stop If

The key echoes to the terminal transcript, appears in argv/report, survives in child env, or Rust prompt failure leaves terminal echo disabled.

Verification System

Claim: Rust now covers hidden prompt setup behavior for the source-checkout cerberus-cli init path that was previously only available through the Node scaffolder.

Falsifier: PTY transcript contains the test key, fake gh lacks stdin bytes, no-key non-TTY succeeds, or workflow scaffolding regresses.

Driver: a local PTY harness runs cerberus-cli init --repo-root tmp/... --report-out tmp/.../report.json with fake gh first on PATH.

Grader: transcript has prompt and success lines but not the key; fake gh records secret set CERBERUS_API_KEY, stdin key, and gh-env.txt as unset.

Delete-First

Requirement questioned: full npm removal is not needed to prove source-checkout prompt parity.

Deleted or simplified: no new prompt flag; no new prompt crate; no Windows prompt promise in this slice.

Only then automated: interactive local setup needs hidden input when env/stdin are unavailable.

Execution

Add focused Rust tests for parser/default behavior and fake-gh child env/stdin invariants.

The TTY echo invariant is covered by live PTY QA, not unit tests.

Noninteractive paths keep existing behavior.

Node remains npm setup/package compatibility until the npm entrypoint strategy is decided.

4 review

Fix blockers and re-review before commit.
