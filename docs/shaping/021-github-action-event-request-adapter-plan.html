Backlog 021

Make GitHub event acquisition a Rust adapter.

Request building moves out of shell first. Hosted API POST, polling, timeout, fail-on-verdict, and GitHub output writes stay in `dispatch.sh` until later parity slices prove them.

Outcome

`cerberus-cli github-action-request` reads a pull-request event payload, skips fork/draft PRs before diff IO, then writes a validated `ReviewRequest.v1` for same-repo PRs.

Chosen Design

Put event interpretation in `cerberus-adapter`; keep CLI as file IO. Share syntax-level diff parsing with `review-local` instead of duplicating it.

Proof Surface

  • Rust tests: same-repo build, fork skip, draft skip, missing head repo skip, malformed diff.
  • CLI QA writes `tmp/github-action-request/review-request.json`.
  • Schema validation proves the emitted request is accepted by the Rust contract.

Stop Conditions

  • Stop if the slice needs network GitHub calls or hosted API polling.
  • Stop if adapter code starts semantic issue/context inference.
  • Stop if replacing public `action.yml` is required for proof.

Verification System

FieldContract
ClaimRust can build a source-agnostic review request from GitHub Actions event data and a checked diff.
FalsifierFork, draft, or missing-head-repo events require diff IO or produce requests; same-repo events fail validation; changed files drift from the diff; shell/API concerns leak into the adapter.
Drivercerberus-cli github-action-request --event ... --diff-file ... --out ...
GraderAdapter tests, CLI QA, schema validation, retirement inventory validation, and fresh critic.
Evidencetmp/github-action-request/
CadenceBefore replacing shell dispatch or making the root action Rust-backed.

Delete-First

Requirement questioned: do we need to port all of `dispatch.sh` now?

Deleted or simplified: no; request construction can be proven without HTTP, polling, or workflow output writes.

Review Focus

Attack public CLI shape, fork/draft parity, trust-boundary parsing, and whether GitHub-specific behavior leaks into the review core.

Residual Risk

This does not prove hosted API compatibility or public action replacement. Those remain later retirement slices.
