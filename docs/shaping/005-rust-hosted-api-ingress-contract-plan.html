Backlog 005 / legacy surface retirement / 2026-06-19

Freeze the legacy hosted API ingress contract in Rust

The next retirement move is a compatibility adapter for POST /api/reviews, not a Rust web server or execution migration.

Target Outcome

Rust can parse the legacy Elixir POST body, preserve exact validation behavior, redact request-scoped tokens from evidence, and emit a checked fixture report.

Design Choice

Keep the API pointer body as adapter input. Do not force it into ReviewRequest.v1, because it lacks diff/files until GitHub acquisition runs.

Proof Surface

Focused Rust tests, CLI fixture reports for accepted and rejected POST bodies, retirement inventory evidence, and unchanged legacy gates.

Stop condition: this pass ends when ingress compatibility is covered. GET /api/reviews/:id, store semantics, API deployment, and review execution remain pending unless their own fixtures land.

Current Evidence

SourceFactImplication
cerberus-elixir/lib/cerberus/api.ex POST accepts repo, integer pr_number, head_sha, optional github_token, optional model. Rust validation should be value-based and ignore unknown fields, matching Plug/Jason behavior.
cerberus-elixir/test/cerberus/api_test.exs Whitespace GitHub token becomes nil; CR/LF token is rejected; unknown fields do not leak into stored response. Fixture output must expose only github_token_present, never the token value.
crates/cerberus-schema/src/lib.rs ReviewRequest.v1 requires a concrete diff and file list. Legacy API POST is a review pointer, not a core request. Mapping to core belongs after GitHub read acquisition.
docs/shaping/legacy-surface-retirement.json elixir-http-api parity is pending. This patch can move parity evidence forward while leaving the surface pending.

Change Shape

  • Add Rust adapter types for hosted API ingress submission and fixture reports.
  • Add exact validation helpers for legacy POST body behavior and accepted 202 response shape.
  • Add cerberus-cli hosted-api-ingress-fixture --body ... --out ... for reviewable accepted/rejected evidence.
  • Add small checked fixtures under fixtures/hosted-api/.
  • Update API docs and backlog 005 retirement inventory with the new evidence and the remaining boundary.

Risks

  • Overclaiming: this is not API retirement. It is ingress parity evidence.
  • Secret handling: reports must not serialize raw GitHub token values.
  • Contract drift: accepted POST body remains a pointer until a Rust server fetches PR context and diff.
