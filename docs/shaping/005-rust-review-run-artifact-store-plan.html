BACKLOG 005 / ELIXIR VERDICT STORE RETIREMENT SLICE

Persist review-run artifacts at the adapter edge, and replay only schema-valid receipts.

Rust already owns ReviewRunArtifact.v1 aggregation and replay consistency. This slice adds the small storage boundary needed by local, CI, and Sprites callers without moving legacy SQLite behavior into the core.

Chosen Design

Add a file-backed ReviewRunArtifactStore in cerberus-adapter. It writes artifacts under review-runs/<run_id>.json, rejects unsafe run IDs, validates before persisting, and validates again after replay.

Quality System

Risk tier: storage boundary for review evidence. Proof: focused adapter tests, schema replay validation, retirement inventory validation, aggregate Rust gates, and fresh critic on the diff plus this oracle.

Standards

Core remains semantic. Adapter owns filesystem IO. No SQLite dependency, no duplicated aggregation logic, and no caller ledger semantics.

Acceptance

A valid review run can be persisted and loaded unchanged; tampered, malformed, or path-escaping artifacts fail closed.

Verify

cargo test -p cerberus-adapter artifact_store, schema validation, retirement inventory validation, and workspace gate.

Stop If

The implementation duplicates verdict aggregation, stores outside the root, claims full Elixir store parity, or weakens existing gates.

Verification System

Claim: Rust has a minimal, source-agnostic persisted review-run receipt path.

Falsifier: malformed JSON, inconsistent stats/cost/verdict, or unsafe run IDs can be stored or replayed as valid evidence.

Driver: generate a real core artifact, persist it, reload it, mutate pre-persist and post-persist JSON, and validate all failure modes.

Grader: valid artifact round trips byte-semantically through public schema types, while tampering fails through schema or adapter errors.

Delete-First

Requirement questioned: full legacy SQLite parity as the first Rust store step.

Simplified: JSON artifact receipts are enough for agent workflows, local QA, and Sprites handoff.

Deferred: hosted API persistence and any database compatibility decision stay pending until API cutover proof exists.

Execution

1 boundary

Add the store trait and file implementation under crates/cerberus-adapter/src/artifact_store.rs, exported from the adapter crate.

No core or CLI ownership change unless the tests prove the boundary is insufficient.

2 tests

Cover valid round trip, unsafe run IDs, malformed pre-persist artifacts, corrupt replay JSON, and tampered persisted artifacts.

Use public Rust review generation for the happy path.

3 docs

Update architecture and the legacy retirement inventory to record the partial store replacement evidence without claiming full parity.

The Elixir store remains pending until hosted/API persistence is proven.

4 review

Fresh critic attacks boundary placement, unsafe paths, replay validation, parity claims, and overengineering.

Fix blockers before commit.
