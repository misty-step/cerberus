Backlog 010

Peer Harness Command Profiles

Make Pi, Goose, OpenCode, and OMP executable through the Rust command adapter as validated profile data, without live provider calls or core shell semantics.

Date: 2026-06-18

Branch: shape/rust-review-engine-backlog

Stop: if a profile cannot be validated without invoking paid models.

Target Outcome

Cerberus has a checked-in `PeerHarnessCommandProfiles.v1` fixture for Pi, Goose, OpenCode, and OMP. Each profile names command, static args, timeout, env requirements, output contract, and unsupported containment boundaries.

Design

Add schema types in `cerberus-schema` and conversion helpers in `cerberus-adapter` that build `CommandHarness` from profile data. Keep `cerberus-core` unchanged.

Acceptance Oracle

  • `cargo test --workspace peer_harness_command_profiles`
  • `cargo test --workspace peer_harness_profile_builds_command_harness`
  • `cargo run --locked -p cerberus-cli -- validate fixtures/harnesses/peer-command-profiles.json`
  • Full Rust, shell, Node, and legacy Elixir gates remain green.

Profile Shape

{
  "schema_version": "peer-harness-command-profiles.v1",
  "observed_at": "2026-06-18",
  "profiles": [{
    "harness_id": "opencode",
    "command": "cerberus-peer-harness",
    "args": ["--harness", "opencode"],
    "timeout_ms": 300000,
    "env_required": ["OPENROUTER_API_KEY"],
    "output_contract": "reviewer_artifact_file",
    "peer": {
      "command": "opencode",
      "args_template": ["run", "--format", "json", "--model", "openrouter/{model}", "--agent", "reviewer", "{prompt}"],
      "prompt_mode": "argv_message"
    },
    "unsupported": [
      "daemonized children outside the process group",
      "unbounded live output files",
      "paid provider calls without an explicit eval budget"
    ]
  }]
}

Boundaries

  • No provider credentials or paid calls.
  • No shell execution in core.
  • No prompt semantics or model ranking yet.
  • No sandbox claim beyond process-group cleanup.

Files

  • `crates/cerberus-schema/src/...` for packet schema.
  • `crates/cerberus-adapter/src/...` for profile-to-command conversion.
  • `fixtures/harnesses/peer-command-profiles.json` for Pi/Goose/OpenCode/OMP.
  • `backlog.d/010-...md` and shaping docs for receipt.

Critic Focus

  • Profile validation catches duplicates, empty args, bad timeouts, missing env names.
  • Command profile conversion cannot smuggle shell behavior into core.
  • Docs make live execution and containment limits explicit.
