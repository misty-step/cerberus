CERBERUS / RUST REVIEW CORE

Evaluate harness and model pairs before promoting reviewer defaults.

Build a matrix runner that tests Pi, Goose, OpenCode, and OMP against current coding models on Cerberus reviewer tasks. The winner is not the loudest benchmark; it is the pair that emits valid reviewer artifacts, finds seeded issues, avoids false positives, and stays inside cost and latency bounds.

CHOSEN DESIGN

Use a Rust-side eval bridge: fixtures plus harness probes plus model catalog snapshot plus HarnessModelEvaluationReport.v1.

QUALITY SYSTEM

Standards: artifact validity, evidence discipline, reviewer quality, fair harness comparison, dated model facts.

Proof: eval runner output, schema validation, transcripts, cost and latency records, model drift report.

Premise source: explicit user request in this thread plus live repo and provider evidence. No preexisting standalone artifact.

Plan status: local smoke runner implemented; live paid model adapters remain future work.

STANDARDS

Current-source model facts, fixed eval tasks, common output schema, no prompt hacks per harness.

ACCEPTANCE

A report can rank harness/model pairs for Cerberus review work and block stale defaults.

VERIFY

cerberus-cli eval-harness, schema validation, fixture grades, transcript review.

STOP IF

No reviewer artifact schema exists, provider facts conflict without live probes, or a public leaderboard is treated as acceptance.

EVALUATION FLOW

01

Freeze tasks

The local smoke suite checks in clean, seeded-bug, prompt-injection, long-context, schema-hostile, and degraded tasks before provider spend.

Prevents benchmark drift and prompt cherry-picking.

02

Probe harnesses

Run the same task contract through Pi, Goose, OpenCode, and OMP. Capture structured failures instead of retry folklore.

Separates model quality from harness brittleness.

03

Grade artifacts

Validate schema, grade seeded findings, count false positives, inspect evidence quotes, and record cost/latency.

Makes reviewer quality falsifiable.

04

Promote candidates

Only report winners become candidate ReviewConfig.v1 entries and Daedalus promotion packets.

Keeps model churn out of production defaults.

cargo test --workspace harness_model_eval

RISKS

Provider catalogs drift, public benchmarks overfit, paid model calls make CI flaky, harness-specific tools make comparisons unfair, and stale local configs can be promoted by accident.

CRITIC FOCUS

Attack the eval for fake fairness: hidden prompt differences, ungraded false positives, no cost ceiling, output schema repair by humans, or model facts copied from stale memory.
