BACKLOG 005 / ELIXIR VERDICT STORE RETIREMENT

Prove hosted API review artifacts can land in the Rust file store.

The Rust artifact store exists, but the retirement inventory still lacks hosted/API persistence proof. This slice lets completed hosted API payloads carry a validated ReviewRunArtifact.v1 and persists it only when an operator explicitly provides a store root.

Chosen Design

Keep protocol interpretation in cerberus-adapter. Keep filesystem writes in cerberus-cli. Reuse FileReviewRunArtifactStore; do not add SQLite, a server, or a new runtime layer.

Quality System

Risk tier: substantive adapter/CLI behavior. Proof: unit tests, fake-server CLI test, fixture smoke, inventory validation, full gates, and fresh critic.

Standards

Validated artifacts only, explicit store root only, no action output drift, no core I/O, and no broad API-server rewrite.

Acceptance

A completed hosted API response with review_run_artifact can be persisted and replayed from review-runs/<run_id>.json.

Verify

cargo test -p cerberus-adapter hosted_api, CLI fake server, fixture smoke, schema validation, and retirement validation.

Stop If

The change needs live API credentials, SQLite compatibility, provider calls, action output changes, or semantic parsing of free-form API prose.

Verification System

Claim: Hosted/API dispatch can persist a schema-valid Rust review artifact without the Elixir store.

Falsifier: invalid artifacts persist, completed payloads lose artifacts, missing requested artifacts pass silently, or GitHub Action outputs drift.

Driver: adapter fixture, CLI fake server, and hosted-api-dispatch-fixture --artifact-store.

Grader: artifact validates, store replay matches, outputs stay review-id/verdict, and inventory validator accepts the updated receipt.

Delete-First

Requirement questioned: whether a Rust API server or SQLite bridge is required now.

Simplified: no new server or DB; reuse the completed-poll payload plus the existing immutable file store.

Deferred: SQLite compatibility and full Elixir API retirement remain separate decisions.

Execution

1 parse

Extend hosted dispatch decisions with an optional validated artifact from completed poll bodies.

Invalid embedded artifacts are invalid hosted API responses.

2 persist

Wire explicit store roots in CLI dispatch and fixture paths, using FileReviewRunArtifactStore.

Missing artifacts fail only when persistence was requested and the hosted review completed.

3 prove

Add fake-server and fixture smoke evidence that replays the persisted artifact.

No live credentials or provider budget involved.

4 retire

Update the retirement inventory to record hosted/API persistence fixture evidence while keeping Elixir store deletion pending.

The slice advances parity; it does not claim full store retirement.
