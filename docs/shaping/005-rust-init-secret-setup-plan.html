BACKLOG 005 / RUST-ONLY SCAFFOLDER

Move noninteractive GitHub secret setup into the Rust CLI without exposing the API key.

The next durable step is cerberus-cli init: compose workflow scaffolding with gh secret set CERBERUS_API_KEY using stdin. This advances Node retirement while preserving the Node wrapper for its remaining hidden-TTY prompt/package boundary.

Chosen Design

Add init beside init-workflow. It reads CERBERUS_API_KEY or --api-key-stdin, writes/preserves the workflow, pipes the secret to gh secret set, and emits a redacted report.

No secret in argv, report JSON, docs, or test fixtures. No new dependency. No real GitHub mutation in tests.

Quality System

Risk tier: high-stakes CLI because it handles secrets. Proof: fake-gh live QA, focused tests, secret-redaction inspection, aggregate gates, and fresh critic.

Standards

Rust owns durable setup logic. Compatibility remains explicit. Existing workflow overwrite protection survives.

Acceptance

cerberus-cli init creates or preserves the workflow, configures the secret through gh stdin, and fails closed when no key is available.

Verify

Focused Rust tests, scratch CLI run with fake gh, report inspection, inventory validation, Node syntax, shellcheck, and full repo gates.

Stop If

The API key appears in an argv string, report, committed fixture, shell history, or test output; or if the command would mutate real GitHub during QA.

Verification System

Claim: Rust owns workflow plus noninteractive secret setup for local/source-checkout setup.

Falsifier: fake gh sees no stdin, sees the wrong secret name, report leaks the secret, or no-key mode silently succeeds.

Driver: printf '%s' test-secret-value | env CERBERUS_API_KEY=parent-secret-value PATH="$PWD/tmp/.../bin:$PATH" cargo run --locked -p cerberus-cli -- init --repo-root tmp/... --api-key-stdin --report-out tmp/.../report.json.

Grader: generated workflow matches template, fake gh records secret set CERBERUS_API_KEY, secret bytes only appear in ignored fake-gh capture, and report says configured.

Delete-First

Requirement questioned: full NPM binary retirement is a packaging problem, not needed to prove Rust secret behavior.

Deleted or simplified: no interactive hidden prompt in this slice; no dependency; no real GitHub API call in QA.

Only then automated: agent/noninteractive setup needs a Rust path that handles secrets without Node.
