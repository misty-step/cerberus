BACKLOG 005

legacy surface retirement

RETIREMENT CONTROL PLANE

Make legacy deletion prove itself before any useful donor behavior is removed.

Add a machine-checked inventory for Elixir, action, CLI, defaults, prompts, and historical artifacts so future Rust parity work can retire surfaces in small, reversible commits.

CHOSEN DESIGN

Add `LegacySurfaceInventory.v1`, a checked JSON inventory, a CLI validator, and active-doc links to the inventory. Do not delete Elixir yet.

PROOF

Schema unit tests, `cerberus-cli validate-retirement`, generic schema validation, full Rust tests, and legacy gates.

STOP IF

A surface is marked retired without a commit, a delete/archive decision lacks parity evidence or rollback, or docs imply Elixir is the long-term engine.

KEEP NOW

GitHub Action dispatch, hosted API compatibility, Node scaffolder, defaults, and persona files stay while callers still depend on them.

PORT NEXT

Review orchestration, aggregation, dedupe, cost, stores, prompt rendering, tools, and local CLI behavior move behind Rust fixtures before deletion.

ARCHIVE LATER

Historical walkthroughs and legacy evidence remain reference material until the inventory records an archive commit and rollback path.

IMPLEMENTATION FLOW

01

Inventory schema

Add a Rust schema for retirement entries: current responsibility, decision, parity status, evidence, commit, rollback, and next action.

Prevents vague retirement prose.

02

Checked table

Check in a dated inventory covering Elixir modules, root action, dispatch, scaffolder, defaults, prompts, templates, and historical docs.

No deletion in this slice.

03

CLI validation

Add `validate-retirement` and generic `validate` support so the table is part of normal Rust gates.

Turns backlog doctrine into a command.

04

Active docs

Update README, architecture, docs index, and backlog receipt to name Elixir as compatibility, not the target engine.

Reduces source-of-truth drift.
