Backlog 006 / No-Spend Report Provenance

Make selected eval reports self-describing before durable automation consumes them.

The paid full-suite provider rerun remains blocked on explicit budget acknowledgement. This slice closes the known no-spend provenance gap: selected reports should state the exact harnesses, models, and tasks they represent instead of relying on output paths and cell inspection.

Chosen Design

Add optional typed metadata. `EvalReportSelection` carries concrete selected harness, model, and task ids on eval, readiness, and budget reports.

Keep full-report JSON backward compatible by omitting the field when no selector was requested.

Stop rule: do not run provider-backed live eval without `CERBERUS_PEER_HARNESS_PROVIDER_BUDGET_ACK`.

Outcome

Selected `eval-harness`, `eval-readiness`, and `eval-budget` outputs name the exact selected dimensions and validate that every listed id appears in the report cells.

Standards

Schema owns report invariants. Core owns report construction. CLI only derives metadata from already-filtered inputs. No new storage, planner, or provider spend.

Proof

Focused red tests, selected CLI QA, schema validation, full repo gates, and a fresh artifact-only critic.

Verification System

Claim: selected reports are durable, unambiguous artifacts for staged harness/model reruns.

Falsifier: metadata can list cells not represented in the report, omit represented cells, or selected CLI output still has no selection field.

Driver: selected no-spend CLI runs for `goose` + `z-ai/glm-5.2` + `clean-no-finding`, plus schema/core/CLI tests.

Evidence: `tmp/evals/subset-metadata-2026-06-19/` reports and validation transcripts.

Critic Focus
