Harden Live Peer Profiles Before the Next Provider Spend

Target outcome: Pi, OpenCode, and OMP provider profiles emit raw assistant text that the strict Cerberus marker parser can inspect, and local Pi/OMP extensions cannot poison eval runs.

Chosen design: keep prompt-file transport and strict artifact parsing, but disable local extension/rule surfaces for prompt-contained eval cells and stop requesting JSON event streams from CLIs whose output is not a single marked artifact.

Proof Surface

No spend: profile/schema validation, execution plans, rendered prompt fixture, and focused Rust tests.

Residual: actual quality still requires a budget-approved live rerun.
