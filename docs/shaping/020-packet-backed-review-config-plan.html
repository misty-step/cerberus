Backlog 020

Run measured packets without hand extraction.

Packet-backed review execution is a workflow bridge, not approval. Review commands may use the embedded config from a validated sandbox packet; production defaults still do not change.

Outcome

`review` and `review-local` accept `--config-packet`, validate the packet, and run the embedded `ReviewConfig.v1` through the existing Rust core.

Chosen Design

Reuse `ReviewerConfigPacket.v1`; add no schema and no install command. `--config` and `--config-packet` are mutually exclusive.

Proof Surface

  • Focused parser/config-source tests.
  • CLI QA for fixture review and local diff review.
  • Negative QA for conflicting config inputs.
  • Full Rust, shell, Node, and legacy Elixir gates.

Stop Conditions

  • Stop if this implies production approval or default mutation.
  • Stop if packet validation is bypassed.
  • Stop if a new schema appears necessary for this bridge.

Verification System

FieldContract
ClaimA validated sandbox packet can drive Rust review commands directly.
FalsifierInvalid packets run, conflicting config inputs are accepted, defaults mutate, or emitted artifacts fail validation.
Drivercerberus-cli review ... --config-packet ... and cerberus-cli review-local ... --config-packet ...
GraderArtifact validation, parser assertions, negative CLI exit, and fresh review.
Evidencetmp/reviewer-config/packet-review/ and tmp/reviewer-config/packet-local-review/
CadenceAfter packet generation, before measured configs are used by callers.

Acceptance Commands

cargo run --locked -p cerberus-cli -- review \
  --fixture fixtures/review-request/clean.json \
  --config-packet fixtures/reviewer-config-packets/daedalus-sandbox-reviewer-config.json \
  --out tmp/reviewer-config/packet-review

cargo run --locked -p cerberus-cli -- review-local \
  --diff-file fixtures/local-review/local.diff \
  --config-packet fixtures/reviewer-config-packets/daedalus-sandbox-reviewer-config.json \
  --out tmp/reviewer-config/packet-local-review
