Backlog 012

Peer Harness Prompt and Transcript Fixtures

Render the deterministic review prompt and parse a local transcript fixture into a validated `ReviewerArtifact.v1`, without invoking peer CLIs or providers.

Date: 2026-06-18

Branch: shape/rust-review-engine-backlog

Stop: if parsing requires semantic inference instead of exact artifact extraction.

Target Outcome

`cerberus-peer-harness` can write the prompt it would send to Pi/Goose/OpenCode/OMP and can parse a fixture transcript containing exactly one marked reviewer artifact JSON block.

Design

Keep prompt rendering and transcript parsing in the runner. Reuse a core-owned public artifact acceptance function so parsed artifacts are checked against reviewer identity, model, findings, and exact request coverage.

Acceptance Oracle

  • `cargo test --workspace peer_harness_runner`
  • `cargo test -p cerberus-cli --test peer_harness_command`
  • Manual runner proof with `--prompt-output` and `--transcript` validates the emitted artifact.
  • Full Rust, shell, Node, and legacy Elixir gates remain green.

Transcript Contract

CERBERUS_REVIEWER_ARTIFACT_JSON_BEGIN
{
  "schema_version": "reviewer-artifact.v1",
  "reviewer_id": "...",
  "status": "completed",
  "verdict": "WARN"
}
CERBERUS_REVIEWER_ARTIFACT_JSON_END

Still Not Live

No peer CLI invocation, model call, OpenRouter request, or transcript interpretation. This slice only renders deterministic input and parses exact marked JSON from a checked-in fixture.

Files

  • `crates/cerberus-core/src/lib.rs` for shared artifact acceptance.
  • `crates/cerberus-cli/src/peer_harness.rs` for prompt and parser flags.
  • `fixtures/harnesses/peer-transcript-with-finding.txt` for parser proof.
  • `docs/shaping/peer-harness-protocol-runner.md` and backlog docs for receipt.

Critic Focus

  • Does transcript parsing avoid semantic heuristics?
  • Can a mismatched transcript artifact escape core acceptance?
  • Do docs make fixture parsing distinct from live review execution?

Next Slice Boundary

After 012, the next live-adjacent work is a no-provider-cost command transcript capture path or a fixture-backed router/local-review replay. Paid model calls remain gated by eval budget and live harness/model evidence.
