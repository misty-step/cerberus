BACKLOG 005 / RUST-ONLY SETUP SURFACE

Retire the unpublished Node scaffolder instead of carrying a second setup engine.

Rust now owns workflow scaffolding, hidden TTY prompting, stdin/env secret input, and gh secret set. The checked npm package is unpublished, so keeping bin/cerberus.js would preserve duplicate behavior without a live consumer contract.

Chosen Design

Delete bin/cerberus.js and package.json. Remove active docs and CI references to npx and node --check. Keep historical walkthroughs as archive material unless they are active verification docs.

Quality System

Risk tier: substantive compatibility deletion. Proof: npm registry 404, active-doc grep, Rust setup QA, retirement inventory validation, CI workflow syntax, aggregate gates, and fresh critic.

Standards

One setup implementation. Rust owns current operator behavior. No active gate depends on a deleted compatibility shim.

Acceptance

The repository has no active npm package or Node scaffolder surface, while documented setup still works through cerberus-cli init.

Verify

cargo test -p cerberus-cli init_workflow, fake-gh CLI QA, workflow YAML parsing, retirement validation, and grep for active Node references.

Stop If

The npm package is published, active docs still recommend npx, CI still runs npm test, or Rust setup QA regresses.

Verification System

Claim: Cerberus setup no longer needs the Node scaffolder.

Falsifier: a published npm package exists, an active workflow gate or doc still references the deleted files, or Rust init fails the workflow/secret setup path.

Driver: npm registry probe, active-doc grep, Rust unit tests, fake-gh setup QA, YAML lint, and retirement inventory validation.

Grader: package is 404, grep has no active references outside historical docs/receipts, Rust QA proves workflow and secret behavior, and gates pass.

Delete-First

Requirement questioned: npm compatibility for an unpublished package.

Deleted or simplified: remove Node setup implementation and npm metadata instead of building a native npm distribution now.

Only then automated: add native package distribution later only if a real consumer or release channel requires it.

Execution

1 evidence

Record the npm 404 and inspect all active references to bin/cerberus.js, package.json, npx, and node --check.

Historical walkthroughs can remain if they are clearly archival.

2 delete

Delete Node package files and remove active CI/docs/gate references.

Do not touch Promptfoo eval workflows; they use Node as an external eval tool, not as Cerberus scaffolder runtime.

3 prove

Run Rust setup tests and CLI QA with fake gh, then validate the retirement inventory and workflow YAML.

The live setup surface is the Rust CLI.

4 review

Fresh critic attacks compatibility deletion, active docs, gate coverage, and Rust-only direction.

Fix blockers before commit.
