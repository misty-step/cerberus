BACKLOG 004

daedalus reviewer config promotion

DAEDALUS / CERBERUS HANDOFF

Accept measured reviewer configs as packets, not vibes or deploy authority.

Define a sandbox-safe packet contract, validate config integrity, and produce a dry-run baseline comparison before any reviewer defaults can change.

CHOSEN DESIGN

Add `ReviewerConfigPacket.v1` and `ReviewerConfigImportReport.v1` schemas. Keep validation and dry-run comparison in Cerberus; leave Daedalus experiments and approvals outside this repo.

PROOF

validate-reviewer-config, import-reviewer-config --dry-run, schema validation, a checked sandbox packet, and a checked import report fixture.

STOP IF

The importer mutates defaults, treats G3-pending packets as production-approved, or needs Daedalus runtime code to validate a packet.

DAEDALUS OWNS

Research runs, score evidence, G1-G5 approvals, prompt packets, and launch packet export.

CERBERUS OWNS

Packet validation, config hash checks, baseline comparison, rollback metadata, and import refusal rules.

THIS SLICE

Sandbox-only import mechanics. No production defaults change and no Daedalus experiment runner enters Cerberus.

IMPLEMENTATION FLOW

01

Packet schemas

Add packet and dry-run report schemas with benchmark identity, score distribution, cost envelope, harness/model metadata, hashes, promotion gate, and rollback fields.

Makes handoff reviewable before import.

02

Core dry-run

Validate packet integrity, run current baseline and candidate config against a clean fixture, and emit an import report with reviewer/model deltas.

No filesystem mutation.

03

CLI commands

Add `validate-reviewer-config` and `import-reviewer-config --dry-run` with optional baseline/fixture overrides.

Matches the backlog oracle.

04

Fixtures and docs

Check in a sandbox Daedalus packet, expected dry-run report, and handoff rules for promotion, rollback, and rejection.

Keeps G3 pending explicit.
