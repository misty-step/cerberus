BACKLOG 005 / HISTORICAL ARTIFACT RETIREMENT SLICE

Index historical walkthroughs before any archive move.

The historical walkthrough surface is intentionally not runtime parity work. This slice makes the evidence navigable and states the archive policy before future cleanup can move or delete files.

Chosen Design

Add a plain Markdown archive index under docs/walkthroughs/. Point docs and the retirement inventory at it. Do not move historical files in this slice.

Quality System

Risk tier: docs/source-of-truth cleanup. Proof: file inventory, link checks by path existence, retirement validation, full gates, and fresh critic for accidental active-surface claims.

Surface

docs/walkthroughs, artifacts, and walkthrough hold prior issue and PR evidence.

Decision

Keep as reference material. Reject as current runtime surface unless a backlog item explicitly cites one file as acceptance evidence.

Acceptance

An index lists current groups, exact counts, policy, and rollback rule without deleting or relocating files.

Stop If

The slice tries to rewrite history, remove receipts, or describe historical Python/Shell material as current architecture.

Verification System

Claim: historical evidence is indexed and no longer an ambiguous retirement action.

Falsifier: files are moved without an archive commit, counts are wrong, active docs still leave readers guessing where old receipts live, or inventory says the surface is complete without an index.

Driver: find current historical files, add the index, validate retirement JSON, and run gates.

Grader: every named root exists, current file counts match, and next action changes from "create index" to maintaining or future archive commit discipline.

Delete-First

Rejected work: moving dozens of evidence files without a release archive tag.

Useful minimum: an index that lets maintainers understand which files are evidence, transcripts, PR bodies, and raw media.

Future move: allowed only as a separate archive commit with rollback path preserved in the retirement inventory.

Execution

1 audit

List current files under docs/walkthroughs, artifacts, and walkthrough.

Counts are evidence for the index.

2 index

Add the archive index with policy, categories, and path lists.

Keep it factual; do not summarize every historical claim.

3 wire

Update docs index, README historical note, retirement inventory, and backlog receipt.

Keep the runtime parity status intentionally rejected.

4 verify

Run retirement validation, workspace gates, and fresh read-only critic.

Any actual archive move is a later slice.
