BACKLOG 003

thinktank review decommission migration

CERBERUS / HISTORICAL DONOR ONLY

Freeze one ThinkTank review run, map its durable concepts, and replay it as a Cerberus artifact without calling ThinkTank.

The compatibility importer lives in `cerberus-adapter`; `cerberus-core` remains the review engine boundary and takes no runtime dependency on ThinkTank semantics or CLI launch behavior.

CHOSEN DESIGN

Use the historical `tmp/review-pr-289` run as a compact frozen fixture: context, plan, manifest, and controlled reviewer summaries become one validated `ReviewRunArtifact.v1`.

PROOF

cargo test --workspace thinktank_migration plus `cerberus-cli validate` on the converted artifact and an `rg "thinktank" crates/ src/` no-runtime-dependency audit.

STOP IF

Conversion requires shelling out to ThinkTank, parsing reviewer prose as semantic truth, or placing ThinkTank-specific concepts in `cerberus-core`.

SOURCE RUN

`review-pr-289` is a degraded historical review/default run with trace and scout completed, proof timed out, and no coverage or degrade-policy artifact because the run predates those files.

MIGRATION SURFACE

Manifest, review context, route plan, reviewer outputs, and run metadata map to public Cerberus schemas. Pi launch, prompt files, auth homes, and traces are rejected or retained only as eval provenance.

CORE BOUNDARY

Cerberus owns review contracts and artifacts. ThinkTank remains donor material; production review execution must not call its CLI or depend on its runtime layout.
