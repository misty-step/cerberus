Backlog 006 / Provider Eval Rerun Control

Make expensive harness/model evals subsettable before the next live run.

The full provider-backed matrix is blocked by budget acknowledgement. The runner should still let an operator rerun hardened cells or staged slices with the same report schemas, readiness checks, and budget estimate path.

Quality System

Outcome: `eval-harness`, `eval-readiness`, and `eval-budget` accept the same `--harness`, `--model`, and `--task` selectors.

Proof: filtered offline eval, no-ack readiness, filtered budget estimate from a full readiness report, schema validation, focused tests, full gates, and fresh critic.

Stop rule: do not spend provider budget or claim model ranking/default promotion.

Current State

Backlog 006 has a six-task suite and a 4x4 matrix. The remaining live rerun is 96 cells, but the CLI can only run all cells.

Chosen Design

Add one shared selector parser in the CLI eval module. Filter the loaded suite and matrix before existing core functions run. `eval-budget` also filters an existing readiness report to the same selected cells.

Delete First

Requirement questioned: a new planner command or run manifest.

Deleted or simplified: no schema version, no scheduler, no persisted plan format.

Only then automated: repeated selected reruns need flags on the existing commands.

Verification

Claim: selected cells generate internally valid eval/readiness/budget reports that match the cells an operator intends to run.

Falsifier: unknown selectors silently pass, budget cells do not match readiness cells, or reports imply full-suite coverage.

Compatibility

No selector preserves current all-cell behavior. Repeated flags are additive. Full-suite promotion gates still require full suite coverage through the existing report-to-config validation.

Review Focus

Attack overclaiming, accidental default-promotion weakening, report identity ambiguity, budget-estimate mismatch, and bloated CLI surface.
