Backlog 005 / hosted API acquisition bridge / 2026-06-19

Lift the hosted API pointer into a core Rust review request

The prior slice froze POST ingress. This slice turns that safe pointer plus explicit PR context and diff into ReviewRequest.v1.

Target Outcome

A Rust fixture command produces a schema-valid ReviewRequest.v1 from legacy hosted API body, PR context JSON, and raw PR diff.

Design Choice

Use explicit fixture inputs for acquired GitHub data. Avoid live network, server runtime, storage, or provider execution in this pass.

Proof Surface

Adapter tests, CLI request fixture, schema validation, token-leak grep, retirement inventory evidence, and fresh critic review.

Stop condition: the bridge is complete when it validates the ingress body, rejects head-SHA drift, parses changed files from the diff, and writes a valid core request. HTTP auth, GET status, store lifecycle, deployment, and live GitHub calls remain pending.

Delete-First Check

  • Requirement questioned: do we need a server now? No. The next missing proof is acquisition shape, not a deployed endpoint.
  • Deleted or simplified: no network client, no storage adapter, no config mode, no model selection behavior.
  • Only then automated: one fixture command because it is repeated parity evidence and catches schema/token drift.
