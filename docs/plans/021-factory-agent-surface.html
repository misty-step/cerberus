Factory lane / backlog 014 plus agent-native packaging

Make Cerberus safe to post and easy for agents to invoke.

Cerberus already has the primary CLI handoff: review-diff returns review output on stdout and a verdict-gated exit code. This branch finishes the factory surface around it: posting requires an explicit caller-injected GitHub token, agents get an intent-shaped MCP server, the repo ships a Cerberus skill, and releases run through Landmark.

Chosen Design

Posting: accept --gh-token-file or --gh-token-env; refuse --post without one; pass the token only to gh api.

MCP: stdio JSON-RPC, intent tools over the existing CLI/review core, no REST mirror and no new review logic.

Release: add the standard Landmark semantic-release workflow and config for master.

Standards

Caller-neutral core; GitHub remains a projection adapter. No ambient auth, no baked bot identity, no static reviewer fleet.

Acceptance

Fake GitHub proves explicit-token posting and no-token refusal; MCP exposes 2-3 high-level review tools; skill documents invocation.

Verify

./scripts/verify.sh, MCP stdio smoke, fake-gh auth log, generated plan/skill docs inspected, final git status clean.

Stop If

If a required GitHub App registration secret is needed, document the consumer step instead of baking token minting into Cerberus.

Execution Plan

1 Safety

Add explicit posting token flags and plumb them to GithubClient. Refuse --post when neither token source is provided.

Proof: fake-gh post with token succeeds; fake-gh post without token exits non-zero and writes no post result.

2 Agent Surface

Add cerberus mcp using stdio JSON-RPC. Tools stay intent-shaped: review a git range, render an artifact, and validate an artifact/request pair.

Proof: protocol initialize, tools/list, and tools/call fixture review smoke under target/cerberus/mcp-*.

3 Skill

Add a shipped skill that tells an agent when to use CLI vs MCP, how to gate on exit codes, and how to avoid posting without explicit token injection.

Proof: skill file exists in .agents/skills/cerberus-review/ and points to repo gate + artifacts.

4 Landmark

Add .github/workflows/release.yml and .releaserc.json using Landmark's semantic-release adoption path for master.

Proof: workflow/config present and verify still passes.
