Context Packet

PR review tracer bullet

Build the smallest path where Cerberus can acquire a PR or git-range, emit a valid ReviewRequest.v1, run the existing harness, and render a durable review artifact.

Chosen design: add request producer commands, not posting or webhooks. The current review harness remains the execution boundary.

Proof Surface

  • ./scripts/verify.sh
  • cerberus request git-range --base origin/master --head HEAD
  • cerberus review --harness fixture --fixture-output fixtures/harness/pass-review.txt over the generated request
  • Evidence in target/cerberus/

Current State

  • Master contains the Rust Cerberus restart.
  • OpenCode is the default substrate; OMP is fallback.
  • Fixture, OpenCode, and OMP harness paths already smoke in the gate.

Stop Conditions

  • Generated request cannot pass existing validation.
  • Diff parsing loses changed-file anchors.
  • Implementation needs GitHub posting to prove acquisition.

Mode Boundary

This is a manual Mode A CLI path. A recurring PR-ready loop belongs in Bitterblossom or another event plane after the manual path proves useful.

Implementation Shape

cerberus request git-range --base <ref> --head <ref> --out <request.json>
cerberus request pr --number <n> --out <request.json>

The producer writes JSON only. Existing review and render commands execute and display the review.

Research Notes

  • OpenCode docs expose run with --format json, --file, --dir, and --attach.
  • GitHub CLI docs expose pr view --json and pr diff --patch/--name-only.
  • Local CLI help matches the documented surfaces.
