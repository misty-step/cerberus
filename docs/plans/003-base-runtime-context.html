Ticket 003Rust CLIContext Truth

Base and runtime context without capability lies

Goal: let Cerberus review with diff-only, head checkout, base+head checkouts, and explicit local probes while the artifact can only claim evidence the request and harness actually made available.

Chosen Design

  • Extend existing ReviewRequest.v1, producer options, and RunWorkspace. No new orchestration engine.
  • Add caller-facing base acquisition through --base-workspace; git-range already has --base, PR gets baseRefOid.
  • Represent local runtime as explicit request targets plus policy.allow_local_runtime. Validation rejects targets without policy.
  • Run local probes before the review substrate, with sanitized env, workspace cwd, timeout, and transcript artifact injection.

Why It Wins

  • It keeps Cerberus a thin harness wrapper around OpenCode/OMP, not a workflow platform.
  • It makes the dangerous part, runtime execution, opt-in and visible in the schema.
  • It lets fake harness fixtures prove the capability matrix without live GitHub or live model calls.

Implementation Slices

  • Schema: add local runtime policy bit and capability derivation.
  • Request: base workspace validation, PR base SHA field, runtime target options.
  • Harness: multi-workspace prep, runtime probes, request artifact injection.
  • Prompt/docs/gate: teach evidence limits and add fixture checks.

Stop Rules

  • Do not implement remote runtime or containers in 003.
  • Do not add predefined subagents or reviewer personas.
  • Do not weaken validation to fit fake harness output.
  • Stop after repeated test failure and re-read the touched module.
