Backlog 004 / Review Kernel Boundary

Make the review kernel the small typed surface.

Cerberus should be easy to extend without threading substrate flags and prompt-schema prose through unrelated layers. The chosen design keeps the existing execution engine, wraps it in a typed ReviewKernel::review(&ReviewRequest, &RunPolicy) -> ReviewRun boundary, and moves OpenCode/OMP knobs into adapter config structs.

Chosen Design

Keep: current Rust harness execution, workspace isolation, runtime probes, artifact validation, fake adapters, and CLI flags.

Change: CLI constructs ReviewSubstrate and RunPolicy, then calls ReviewKernel.

Avoid: a new workflow engine, predefined subagents, generic adapter framework, or model-eval ownership.

Standards

Deep module, small interface. Typed Rust contracts. CLI as adapter. Prompt/schema drift caught by tests. No extra abstraction unless it deletes coupling.

Acceptance

Kernel API owns review execution. Substrate configs own OpenCode/OMP flags. Prompt instructions are checked against Rust artifact shape. Backlog 004 marked shipped.

Verify

cargo test --locked, targeted kernel/prompt tests, live fixture/OpenCode/OMP CLI runs, then ./scripts/verify.sh.

Stop If

The boundary becomes a semantic wrapper, CLI compatibility breaks without need, or review needs product behavior outside the ticket.

Current State

  • src/main.rs directly constructs ReviewHarness from CLI substrate flags.
  • src/harness.rs mixes public harness config, execution, adapter command construction, runtime probes, and parsing.
  • src/prompt.rs manually lists required ReviewArtifact.v1 fields.

Change Shape

  • Add ReviewKernel, RunPolicy, ReviewRun, and ReviewSubstrate types.
  • Move OpenCode/OMP/fixture options into config structs consumed by the kernel.
  • Keep existing command execution private and compatible with fake substrate verification.
  • Add prompt/schema contract tests that enumerate top-level artifact fields and role-linked nested fields.

Verification System

Claim: callers can review through a small typed kernel while substrate specifics stay in adapter configs.

Falsifier: adding a substrate forces prompt changes, CLI-only flags enter kernel structs, or artifact schema drift compiles while prompt prose is stale.

Driver: tests plus live CLI fixture/fake OpenCode/fake OMP review runs.

Evidence packet: target/cerberus/ artifacts, execution plans, transcripts, and final gate output.

Review Focus

  • Thermo: reject wrappers that only rename ReviewHarness.
  • Works critique: preserve CLI/operator behavior and evidence readability.
  • Delete-first: no generic substrate registry until a second real production substrate needs it.
