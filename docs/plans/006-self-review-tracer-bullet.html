Backlog 006 / Self-Review Tracer Bullet · supersedes the shipped pr-review-tracer-bullet slice · scoped after adversarial critique + operator direction

Cerberus reviews its own pull requests.

The whole pipeline is built and fixture-tested: review-pr acquires a PR, the OpenCode harness runs in a disposable worktree with a scrubbed env, the artifact is validated, and post.rs posts an idempotent per-head-SHA summary plus inline comments. It has never met a real model. This bullet drops a real, fully-capable GLM 5.2 agent into that pipe, lets it explore the whole repo autonomously, and posts a genuine review to a real Cerberus PR — the cleanest possible dogfood and the first proof the reviewing brain works at all.

Chosen Design

Empower the agent: full toolset (bash → rg/ast-grep/git log/diff/blame, webfetch, websearch) and no exploration cap — bounded only by wall-clock.

Safety by architecture: the worktree is disposable (writes are thrown away); hard isolation for untrusted PRs is the container's job (ticket 013), not a tool restriction.

Reuse: review-pr → harness (worktree + history, isolated env) → validation → post.rs idempotent summary + inline.

Avoid: subagent orchestration, container runtime, new schema, first-party agent loop.

Standards

One master agent, no static roster. Fully capable: bash, git, rg/ast-grep, web. The diff is a hint; the agent reads the real tree + history. Findings grounded in inspected anchors. Idempotent per-head-SHA posting. Filesystem safety comes from worktree disposability, not tool denial.

Acceptance

A real GLM 5.2 review of the branched vision+groom PR posts a schema-valid summary + ≥1 inline comment on a changed line; a rerun (frozen head) updates, not duplicates; a human confirms every finding's claim — not just its anchor — is real.

Verify

./scripts/verify.sh stays green; the operator-gated live run (--dry-run then --post) posts to a real PR; evidence committed under evidence/self-review-001/.

Stop If

Exploration never converges within the timeout, findings cite lines that don't exist, reruns duplicate comments, or the human read finds confident hallucinations. (Key-exfil is out of scope here — we review our own code; ticket 013 isolates untrusted PRs.)

Load-bearing surprise — the agent is throttled to a shallow review. build_opencode_message (src/prompt.rs:261) caps it at "at most one tool-call round and at most eight targeted file reads… stop tool use and produce the final artifact immediately." That is fixture-era shallow-by-design — the opposite of autonomous exploration. Remove the cap entirely. The agent should roam the full repo and history (this is also how the "prohibitively large diff" problem is solved — explore the tree, don't stuff the diff into the prompt). Tension: more flat context can dilute attention (SWE-PRBench) — but that argues for the agent actively navigating (grep/ast-grep/git) rather than being force-fed context, plus re-anchoring discipline and a human read. Enforced grounding gate is ticket 007.
Security, scoped correctly (after adversarial critique). A critic noted the generated opencode.json denies only edit, leaving bash/network open with OPENROUTER_API_KEY in the child env — an exfil path. That matters for untrusted third-party PRs, where a malicious diff could prompt-inject the agent. It does not gate this bullet: we review our own code, and the worktree is disposable so writes are harmless. The real fix is network-egress isolation + credential handling in the container (ticket 013), which is where untrusted-code isolation belongs — not crippling the reviewer's tools. We keep the critic's other catches: --timeout-seconds (not --timeout), and per-head-SHA idempotency needs a frozen-head PR.

Change Shape

  • 1. Prompt (src/prompt.rs). Remove the 1-round/≤8-read/stop cap in build_opencode_message; instruct autonomous full-repo exploration (use git history, grep, ast-grep; the diff is a starting hint) bounded by wall-clock with "converge and emit one artifact". Add reviewer-craft: prioritize correctness/security/regressions, calibrate severity, prefer few high-signal findings, re-anchor every finding to an inspected line, state what was not inspected. Keep the artifact contract verbatim.
  • 2. Permissive substrate (src/harness.rs). In write_opencode_config stop denying edit and allow the full toolset (or pass --dangerously-skip-permissions) so the agent runs bash/git/web without permission prompts (also avoids the headless external_directory: ask hang). Ensure the worktree exposes full git history.
  • 3. Wiring. --model openrouter/<glm-5.2-slug>, --allow-env OPENROUTER_API_KEY, generous --timeout-seconds (≈1800; the only governor — opencode has no turn/timeout flag).
  • 4. Live path (scripts/verify.sh). Extend the CERBERUS_LIVE_REVIEW_PR block to pass model + allow-env + timeout-seconds.
  • 5. First PR + evidence. Branch the uncommitted VISION.md + backlog 006–013 + archives; open the PR; --dry-run, human-read, then --post; commit evidence/self-review-001/* + posted comment permalinks.

Verification System

Claim: a real, fully-capable GLM 5.2 agent exploring the worktree + history autonomously produces a valid, grounded, useful ReviewArtifact.v1 for a real Cerberus PR and posts it idempotently.

Falsifier: artifact invalid · findings cite nonexistent lines · reruns duplicate · agent never explores beyond the diff · exploration never converges within the timeout.

Driver: the review-pr … --dry-run then --post command, run twice (frozen head).

Grader: ReviewArtifact.v1 validation (machine) + a human read that each finding's claim is real and the transcript shows real exploration + the GitHub diff showing no duplicate threads.

Evidence packet: committed evidence/self-review-001/* (incl. transcript proving git/grep use) + posted PR comment permalinks.

Cadence: once now (proof-of-life); then on every prompt/model change.

Gaps / waiver: automated groundedness enforcement + quality measurement are ticket 007 — here a finding's claim is checked by a mandatory human read on the --dry-run artifact before any --post. Untrusted-PR isolation (network egress + credential handling) is ticket 013. GLM-5.2 validity-rate over N runs is a 006 follow-on.
