CERBERUS RESTART MVP

Deliver one master reviewer with context-adaptive artifacts.

Build the smallest Rust application that accepts source-agnostic review requests, gives a single Cerberus master the available context through OpenCode or OMP, validates the returned artifact, and renders review output. No predefined subagents. No eval product.

Chosen Design

Rust owns: contracts, capabilities, harness execution, validation, receipts, rendering.

OpenCode/OMP own: reviewer judgment, dynamic subagent creation, synthesis.

Default: OpenCode for production-oriented execution; OMP as local fallback.

Stop line: do not implement static reviewer roles or an evaluation matrix.

Standards

Deep public seam: ReviewRequest.v1 -> ReviewArtifact.v1. Runtime context must be truthful. Product path is OpenCode; OMP remains supported; fixture path exists only for deterministic verification.

Acceptance

./scripts/verify.sh passes and leaves artifact, Markdown review, and execution plan evidence under target/cerberus/.

Proof

Structural Rust tests, CLI smoke with fixture harness, substrate execution-plan inspection, and fresh-context critic against diff plus spec.

Current State

The Cerberus checkout is intentionally empty after reset. This plan and spec.md are the new source of truth.

Change Shape

Add a simple Rust CLI/library, request/artifact schemas, OpenCode and OMP harness boundaries, fixture harness, renderer, tests, and verification script.

Stop If

Stop if implementation requires static subagent roles, direct provider orchestration, hosted runtime, GitHub-only coupling, or eval ownership to pass MVP.

Verification System

ClaimCerberus can turn a review request into a validated, renderable artifact while recording available context truthfully.
FalsifierBad request/artifact digests pass, invalid marker output is accepted, inline comments point outside changed files, or ambient secrets enter child env.
Driver./scripts/verify.sh
GraderRust tests, CLI exit code, generated artifact validation, Markdown output inspection, and execution plan assertions.
Evidencetarget/cerberus/artifact.json, target/cerberus/review.md, target/cerberus/execution_plan.json
CadenceBefore closeout and after any harness, schema, or renderer change.
