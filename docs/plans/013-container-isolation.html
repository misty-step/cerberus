Backlog 013 / Isolating Untrusted-PR Review · v3 — reframed via the lethal-trifecta lens

Make the credential worthless, then keep a light box.

Reviewing a consumer's untrusted PR is the lethal trifecta (Willison, 2025): untrusted content + a valuable reachable secret + an egress path. Break any leg and you're safe. Two earlier drafts attacked the secret-leg the hard way (perfectly hide a long-lived key behind a proxy). This one breaks the value of the secret instead — the cheapest robust win, and exactly what Anthropic ships for Claude Code sandboxing: short-lived, narrowly-scoped credentials so theft has near-zero blast radius. A modest container then handles non-credential damage.

Chosen Design

Floor — scoped ephemeral key. Cerberus mints a per-review OpenRouter key capped at ~$0.50 via the provisioning API, hands it to the agent (in-container is fine), and DELETEs it after — crash-safe teardown + an orphan-key sweeper.

Depth — lightweight container. .git-less tree mount, host checkout not mounted, non-model egress blocked. Stops the other damage a worthless key doesn't cover.

Demoted to options: the key-out proxy, managed sandbox, self-microVM. North star: the tool-execution split (C).

Why this dissolves the critic's worst findings. The previous critique killed "egress-allowlist + key in container" because DNS tunnels a CONNECT proxy and the one allowed host (the model API) is itself an exfil sink while the agent holds a valuable key. Make the key worth cents for seconds and both attacks stop mattering: there is nothing worth tunneling out, and a key sent to the model API can't outspend its cap before revocation. We stop trying to win an un-winnable "hide a powerful secret from a shell" game.
The one thing to get exactly right (grounded). OpenRouter keys have a per-key USD cap and programmatic DELETE, but no native TTL (confirmed: provisioning Management API, 2026-06-23). So "worthless if stolen" rests on the cap plus reliable revocation: the orchestrator must delete the key even if the review crashes (a teardown guard and a sweeper that revokes orphaned review keys on the next run). The provisioning key becomes secret-zero — it stays host-side and never enters the container.

Premise resolved

Don't hide an absolute secret — remove its value. Cerberus owns the credential lifecycle (mint/scope/revoke) + a reference container behind the substrate seam. Trusted self-review stays on the local harness, unchanged.

Acceptance

After a review, the minted key is revoked and was capped; a red-team family proves a stolen key is dead and that the agent reached no non-model host and could not touch the host checkout.

Verify

Steal-and-replay probe: exfiltrate the key during the run, try to use it after — must fail (revoked / over-cap). Plus egress log + host-tree digest + artifact validity.

Stop / waiver

If a future provider lacks scoped keys, fall back to the key-out proxy (option B). Never call the local/un-scoped path "untrusted-safe."

Change Shape (phased)

  • M1 — scoped-key lifecycle (the floor). Mint a per-review OpenRouter key with limit via the provisioning API; inject it; DELETE on teardown with a crash-safe guard; an orphan-key sweeper revokes keys left by crashed runs. The steal-and-replay probe proves a leaked key is dead after the run. Testable without a container.
  • M2 — lightweight container (depth). container-opencode substrate: docker run on an internal network, .git-less git archive tree mounted, host checkout never mounted, non-model egress blocked. Covers the damage a worthless key doesn't (other secrets, SSRF, trashing). Red-team family: DNS/endpoint/output/file-write/worktree-escape.
  • M3 — optional upgrades. Key-out proxy (B) for providers without scoped keys; managed sandbox (D) for microVM-grade isolation; both behind the same seam, no schema change.

Trusted self-review (our own code) keeps the local/worktree harness; this is the untrusted path only.

Oracle — executable definition of done

## 1. Scoped-key lifecycle (the floor)
# review mints a key with a USD cap, uses it, DELETEs it.
# STEAL-AND-REPLAY: capture the key mid-run, then after teardown:
curl -H "Authorization: Bearer $STOLEN" https://openrouter.ai/api/v1/chat/completions ...
#   -> must FAIL (401 revoked / 402 over-cap). The minted key GET returns 404/disabled.
# CRASH SAFETY: kill the run mid-review -> the orphan sweeper revokes the key on the next run.

## 2. Container depth
cargo run -- review --request fixtures/requests/redteam-*.json --harness container-opencode ...
#   - non-model egress (curl evil.com) fails; mounted tree has no .git; host checkout digest unchanged.

## 3. Clean review still works + ./scripts/verify.sh green (container path skipped when Docker absent).

Verification System

Claim: a prompt-injected agent on an untrusted PR cannot obtain a credential worth stealing or reach a non-model host; any credential it does capture is capped and revoked by run-end.

Falsifier: the minted key outlives the run (teardown/sweeper failed); the key has no cap; a stolen key still works after the run; the agent reaches a non-model host; the host checkout mutates; a clean review can't produce a valid artifact.

Driver: the steal-and-replay probe + a crash-injection (kill mid-run, assert sweeper revokes) + the red-team container family with egress capture.

Grader: post-run the minted key is dead (API confirms) and was capped; egress log shows model-only; host-tree digest unchanged; artifact validates.

Evidence packet: the key create/delete API transcript, the steal-and-replay result, the crash-sweeper log, egress capture, under target/cerberus/container-*.

Cadence: the key-lifecycle tests always; the container red-team gated on Docker in verify.sh; on every change to the credential/sandbox path.

Gaps / waiver: residual = a stolen key is usable for seconds up to the cap until DELETE lands; accepted. The provisioning key is secret-zero (host-side). Provider lock-in on scoped keys → option B is the fallback.

Adversarial Review (two rounds, folded in)

Round 1 (security critic) killed the egress-allowlist+key-in-container design (DNS bypass, endpoint-as-sink, unwinnable scrub, worktree-as-host-handle). Round 2 (research + premise challenge) showed the deeper fix is to break the value leg of the lethal trifecta — a scoped ephemeral key — which retires most of the round-1 machinery. Residual focus for /deliver: prove crash-safe revocation under SIGKILL/host crash (not just clean exit); confirm OpenRouter's per-key cap is enforced server-side promptly; decide whether M2's container is worth it before any consumer is live, or whether M1 + diff-only-on-no-container suffices to start.
