BACKLOG 001 / EXECUTOR CONTRACT

Make Cerberus safe before it can become automatic review infrastructure.

Deliver hosted verification, trusted substrate binary selection, strict artifact cross-link validation, and adversarial fixtures. This is a hardening slice, not GitHub posting or reviewer-quality work.

Outcome

Cerberus cannot execute a substrate binary selected from hostile parent state or accept malformed artifacts that would later be posted.

Standards

Small Rust surfaces, behavior tests over implementation tests, strict artifact admission, no weakened gates.

Proof

Failing-then-passing adversarial unit tests, ./scripts/verify.sh, and a GitHub Actions workflow running the same gate.

Stop If

CI needs secrets, resolver hardening breaks explicit fixture paths, or artifact strictness rejects valid current fixtures.

Verification System

ClaimCerberus refuses hostile binary resolution and malformed artifact graphs, and the repo gate runs on PRs.
FalsifierA fake opencode earlier in parent PATH executes; an unknown finding_id or fix id passes; two artifact candidates are accepted; PRs lack hosted verification.
DriverRust unit tests, fixture harness smoke, ./scripts/verify.sh, GitHub Actions verify.yml.
Evidencetarget/cerberus/*, workflow check URL, failing-then-passing test names.

Execution Sequence

  1. Add the hosted verification workflow.
  2. Write adversarial tests for resolver, artifact cross-links, and multi-candidate extraction.
  3. Harden resolver, validation, and extraction.
  4. Update spec.md threat model and mark backlog 001 shipped/done only if all oracles pass.
  5. Run the gate, open PR, and get fresh review.

Non-Goals

No review-pr, GitHub posting, base/runtime context tiers, Daedalus receipt bundles, or model/harness evaluation matrix in this slice.
