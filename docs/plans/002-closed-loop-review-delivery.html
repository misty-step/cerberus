BACKLOG 002 / EXECUTOR CONTRACT

Close the loop from a GitHub PR to validated Cerberus review output.

Deliver cerberus review-pr as orchestration over PR acquisition, review execution, artifact validation, dry-run posting plans, and idempotent GitHub projection. Keep ReviewArtifact.v1 caller-neutral; GitHub output is a renderer/poster.

Outcome

An operator can run one command for a pull request and get receipts plus either an exact dry-run plan or GitHub output without duplicate comments.

Standards

Small Rust seams, explicit posting plan, dry-run before writes, idempotency markers, no GitHub-specific mutation of the artifact.

Proof

Unit tests for line mapping and idempotency, fake-gh integration in ./scripts/verify.sh, and an operator-gated live smoke.

Stop If

GitHub write credentials are unavailable for live smoke, inline mapping cannot be proven from the diff, or posting would require hidden side effects.

Verification System

Claimreview-pr can acquire a PR, run a harness, validate the artifact, write receipts, and produce or apply a deterministic GitHub posting plan.
FalsifierA repeated run duplicates output, a stale head SHA posts, an out-of-diff inline comment is posted, dry-run differs from post mode, or failed artifacts publish as successful.
Driver./scripts/verify.sh with fake gh, unit tests for PostPlan.v1, and optional CERBERUS_LIVE_REVIEW_PR=1 live smoke.
Evidencetarget/cerberus/review-pr/*, fake gh transcript, generated post plan, live PR/check URLs when enabled.

Execution Sequence

  1. Document review-pr, posting modes, receipts, and GitHub credential boundaries.
  2. Add PostPlan.v1 with check/status, review comment, and contextual summary operations.
  3. Add diff line mapping so only changed PR lines become inline comments.
  4. Implement review-pr dry-run and posting against a gh-backed adapter with idempotency markers.
  5. Extend ./scripts/verify.sh with fake-gh dry-run/post/idempotency smoke and document optional live self-review.
  6. Run code review, QA, refactor lanes, then squash merge if clean.

Credential Boundary

GitHub Checks write access is not universally available to classic user tokens. The posting plan must name whether it will use Checks API or commit-status fallback, and post mode must fail clearly when the selected operation is unauthorized.

Non-Goals

No hosted worker, no long-running service, no model evaluation, no new static reviewer topology, and no mutation of ReviewArtifact.v1 to fit GitHub.
