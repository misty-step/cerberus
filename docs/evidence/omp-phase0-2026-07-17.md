## Outcome

OMP v17.0.2 now has a pinned, fail-closed Cerberus path. The runtime:

- resolves only an absolute or trusted-path executable and verifies `config/omp-version.json` before model execution;
- invokes `-p --mode json` with the existing isolation flags, a private prompt file, and a private per-run config overlay;
- disables memory embeddings, retries, and model fallback for deterministic fresh-XDG runs;
- keeps prompt, request, and config paths redacted from `ExecutionPlan.args`;
- retains `env_clear()` and copies only explicitly allowed environment into the model process;
- rejects malformed NDJSON, missing or duplicate terminal `agent_end`, missing final assistant state, and `error`/`aborted` terminal stop reasons before reading an emitted artifact;
- still requires the emitted `ReviewArtifact.v1` to pass request-bound validation.

OpenCode remains the default. This phase does not change `ReviewRequest.v1`, `ReviewArtifact.v1`, the kernel seam, or the reviewer topology.

## Security boundary and residual risk

`PI_CODING_AGENT_DIR` was explicitly allowlisted only for these first-party proof runs. It gives the child access to the local provider credential database; it is not ambient inheritance and is not an acceptable untrusted-PR boundary. Phase 5 owns subscription-auth packaging without long-lived secret forwarding.

Phase 0 proves headless lifecycle and current request-bound artifact admission. It does not prove mandatory seat identity/family independence, truthful model-authored receipts, global concurrency, caching, untrusted PR execution, or the OMP default flip. Those remain gated by later phases and ADR/spec/VISION reconciliation in the productization plan.
