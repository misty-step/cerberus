# Grooming Plan — 2026-03-22

## Session Theme: Migration Completion + Cleanup

**Direction locked:** Complete the Elixir migration, achieve full portability, decommission Python.
Three priorities in order: portability → cleanup → quality.

## Strategic Rationale

1. **Elixir engine is code-complete but not operational.** All 21 issues from last groom are closed, but the engine has never been deployed, has no Dockerfile, no CLI, and no local-run story.
2. **Python is dead weight.** 41K LOC of scripts/tests/glue that the 9K LOC Elixir engine fully supersedes. 78% of Python is GHA glue/ceremony (grug analysis).
3. **Portability is the priority.** Cerberus must run via GHA dispatch, locally on any machine, or on any server — NOT coupled to Fly.io/Sprites.
4. **Cleanup before features.** Tidy, refactor, simplify. Then improve review quality.

## Architecture Decision

**Option 3 (Clean Slate) selected.** Extract Engine, add CLI + Dockerfile, delete Python, restructure repo. Target: ~6K LOC Elixir + 60 LOC GHA action.

### Key Architectural Insight (Ousterhout)

Pipeline conflates input acquisition, review execution, and output posting. Fix: extract `Cerberus.Engine.review(diff, context, opts)` that accepts data and returns data. Callers handle I/O.
